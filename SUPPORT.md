# Support

This document explains how to get help and how to request improvements for EquiPilot AI.

## Bug reports
If you believe you found a bug, please:
1. Search existing issues to avoid duplicates.
2. Create a new issue using the **Bug Report** template.
3. Include:
   - What you expected to happen
   - What actually happened
   - Steps to reproduce (including inputs)
   - Relevant logs/errors (remove secrets)
   - Your environment details (OS, Python version, dependency versions)

## Feature requests
For feature requests:
1. Search existing issues first.
2. Create a new issue using the **Feature Request** template.
3. Include:
   - The problem you’re trying to solve
   - The proposed solution (optional)
   - Any example(s) or use cases
   - Priority/impact to your workflow

## Discussions
If you want to ask questions, propose ideas, or discuss design considerations:
- Use GitHub Discussions (if enabled for this repository), or
- Start a topic in GitHub Issues with a clear title and context.

## Contact guidance
For urgent matters related to security or vulnerabilities:
- See [`SECURITY.md`](./SECURITY.md).

For other support requests:
- Provide details in an issue using the templates in `.github/ISSUE_TEMPLATE/`.

Thank you for contributing to EquiPilot AI.

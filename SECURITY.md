# Security Policy

## Supported versions
We support security updates for:
- The latest released version
- The current development branch (main)

If you report a vulnerability that affects only older versions, we will still acknowledge it, but timelines for fixes may vary.

## Reporting vulnerabilities
Please report security vulnerabilities responsibly and privately.

- Open a private security advisory if GitHub is available in your instance, **or**
- Submit a report via the contact guidance in [`SUPPORT.md`](./SUPPORT.md).

## Responsible disclosure process
When we receive a report, we will:
1. **Acknowledge receipt** as soon as possible (typically within a few business days).
2. **Triage and assess impact** (reproducibility, severity, affected components).
3. **Coordinate a fix and release plan** with maintainers.
4. **Publicly disclose** the details only after a patch is available, unless we determine an earlier disclosure is necessary to protect users.

## Response expectations
We aim to respond to reports with severity-based priorities:
- **Critical/High**: Investigation started promptly; mitigation guidance or a fix may be released faster.
- **Medium/Low**: Triaged in due course; fix may be scheduled for the next appropriate release.

If we need clarification to reproduce the issue, we may request additional information from the reporter.

## What to include in your report
Please provide:
- A clear description of the vulnerability
- Steps to reproduce (or a minimal proof of concept)
- Expected vs actual behavior
- Impact analysis (what data/operations are affected)
- Affected version(s) and environment details
- Any relevant logs or screenshots (remove sensitive tokens/secrets)

Thanks for helping keep EquiPilot AI safe.
